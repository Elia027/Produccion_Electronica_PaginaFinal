Nuestro Proyecto: Tamagotchi en PCB

Una mascota virtual electrónica creada desde cero

Idea Original

Inicialmente teníamos en mente crear una consola de videojuegos portátil, pero debido a limitaciones de tiempo y recursos, decidimos enfocarnos en un proyecto más específico pero igualmente desafiante: recrear un Tamagotchi clásico pero con nuestra propia implementación en PCB.

El objetivo era aprender sobre diseño de circuitos impresos, programación de microcontroladores y manejo de interfaces de usuario con pantallas OLED.

Proceso de Desarrollo

Pasos que seguimos para crear nuestro Tamagotchi

Investigación y Planeación

Estudiamos el funcionamiento de los Tamagotchi originales y determinamos los requisitos técnicos para nuestra versión. Definimos las funciones básicas: alimentar, jugar, dormir y monitorear estado.

Diseño Esquemático en KiCad

Utilizamos KiCad para crear el esquemático del circuito, conectando todos los componentes según sus especificaciones y asegurando la compatibilidad entre ellos.

Diseño de la PCB

Convertimos el esquemático en un diseño de placa de circuito impreso, organizando los componentes para minimizar el tamaño y optimizar las rutas de las pistas.

Fabricación de la PCB

Utilizamos la monofab para fresar la placa y crear las pistas del circuito según nuestro diseño. Luego aplicamos soldadura para proteger las pistas.

Ensamblaje de Componentes

Soldamos cuidadosamente cada componente en su lugar correspondiente en la PCB, verificando continuamente que no hubiera cortocircuitos o conexiones incorrectas.

Programación del Microcontrolador

Desarrollamos el código en Arduino IDE para controlar la pantalla OLED, leer los switches, generar sonidos en el buzzer y gestionar la lógica del Tamagotchi.

Pruebas y Depuración

Probamos cada función del Tamagotchi, corrigiendo errores en el código y solucionando problemas de hardware cuando era necesario.

Consejo: La organización y planificación son clave en proyectos de electrónica. Documentar cada paso nos ayudó a identificar y resolver problemas rápidamente.

Diseño Esquemático

Diagrama eléctrico de nuestro Tamagotchi

Conexiones Principales

El esquemático muestra cómo interconectamos todos los componentes:

  • ATmega32U8P-P: Corazón del sistema, con conexiones a todos los periféricos
  • Pantalla OLED: Conectada via I2C (SDA y SCL) para comunicación
  • Buzzer: Conectado a un pin PWM para generar diferentes tonos
  • Switches: Conectados con resistencias pull-up para entrada digital
  • Crystal y capacitores: Proporcionan el reloj de 16MHz para el microcontrolador
  • LED verde: Indicador de alimentación con resistencia limitadora

Diseño de la PCB

Placa de circuito impreso personalizada

Características del Diseño

Nuestra PCB fue diseñada con las siguientes consideraciones:

  • Tamaño compacto: Similar a un Tamagotchi original
  • Doble capa: Para optimizar el espacio y reducir complejidad
  • Pistas anchas: Para manejar adecuadamente la corriente
  • Agujeros de montaje: Para futura carcasa (aunque no la implementamos)
  • Organización lógica: Componentes agrupados por función

Implementación del Código

Programando la lógica del Tamagotchi

Estructura del Programa

El código gestiona todas las funciones del Tamagotchi:

  • Máquina de estados: Controla los diferentes modos (comer, jugar, dormir)
  • Temporizadores: Gestiona el envejecimiento y necesidades de la mascota
  • Gráficos: Dibuja la mascota y la interfaz en la pantalla OLED
  • Entrada: Lee los switches para interacción del usuario
  • Sonido: Genera efectos de audio mediante el buzzer
// Fragmento de código del Tamagotchi
#include <Wire.h>
#include <Adafruit_SSD1306.h>

// Definición de pines
#define BUZZER_PIN 9
#define BUTTON_FOOD 2
#define BUTTON_PLAY 3
#define BUTTON_SLEEP 4

// Estados del Tamagotchi
enum PetState {HAPPY, HUNGRY, BORED, SLEEPY, SLEEPING};

PetState currentState = HAPPY;
int hunger = 50;
int happiness = 70;
int energy = 80;

void setup() {
  // Inicializar pantalla OLED
  display.begin(SSD1306_SWITCHCAPVCC, 0x3C);
  display.clearDisplay();
  
  // Configurar pines
  pinMode(BUZZER_PIN, OUTPUT);
  pinMode(BUTTON_FOOD, INPUT_PULLUP);
  pinMode(BUTTON_PLAY, INPUT_PULLUP);
  pinMode(BUTTON_SLEEP, INPUT_PULLUP);
  
  // Mostrar pantalla de inicio
  displayWelcome();
}

void loop() {
  // Actualizar estado según el tiempo
  updatePetStatus();
  
  // Leer botones
  checkButtons();
  
  // Mostrar estado actual
  displayPetStatus();
  
  delay(100);
}

void feedPet() {
  hunger = min(100, hunger + 30);
  playSound(HAPPY_SOUND);
}

void playWithPet() {
  happiness = min(100, happiness + 25);
  energy = max(0, energy - 15);
  playSound(PLAY_SOUND);
}

Reto superado: La gestión de memoria fue un desafío importante debido a las limitaciones del microcontrolador. Optimizamos el código para usar la menor memoria posible mientras manteníamos todas las funcionalidades.
